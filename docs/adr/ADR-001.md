# ADR-001: Local-First Architecture Decision

**Status:** Accepted

**Date:** 2025-01-01

## Context

SovereignSpec operates on a core thesis: specifications are the durable artifact, code is disposable, and nothing leaves your machine. To uphold this thesis, the architecture must ensure that no inference request, no data point, and no user action ever requires a cloud API call.

Existing Spec-Driven Development tools (notably GitHub's Spec Kit) rely on cloud LLM endpoints for every pipeline step: `/clarify`, `/plan`, `/implement` all call external APIs. This creates three categories of risk:

1. **Data leakage**: Specifications often contain proprietary business logic, API designs, or architectural details. Sending these to third-party LLM providers exposes intellectual property.
2. **Latency and availability**: Cloud LLM endpoints have variable latency and may be unavailable. Spec Kit's workflow halts if the API is down.
3. **Cost**: Per-token billing makes spec-driven development expensive at scale, especially during iterative clarification cycles.

## Decision

SovereignSpec will require zero cloud API calls for any of its core functionality. All LLM inference will run locally via Ollama (llama-cpp compatible). All data storage will be local files (SQLite, ChromaDB, adjacency JSON). The UI will run on localhost.

Specifically:

1. **Inference**: All LLM calls use Ollama's REST API at `localhost:11434`. No model is loaded from a cloud provider. No prompt or spec content is sent over the network.
2. **Vector storage**: ChromaDB runs in embedded mode within the same Python process. All embeddings are computed locally via Ollama's embedding endpoint.
3. **Metadata storage**: SQLite runs as a local file. No database server is required.
4. **Graph storage**: Adjacency JSON is the default format (zero dependencies). Neo4j is optional and, if used, runs locally.
5. **UI**: Next.js development server on localhost. No deployment to Vercel or any cloud platform is required or encouraged.
6. **Integration**: Agent communication is purely filesystem-based. No API integration with any coding agent.

## Rationale

Local-first architecture aligns with SovereignSpec's core value proposition: the spec is the durable artifact and the code is disposable. If the system depends on cloud APIs, then the spec-driven workflow itself depends on external infrastructure — making the specs less durable, not more.

The local-first constraint also enables air-gap operation, which is a requirement for classified or proprietary codebase environments. No network connectivity is needed after initial tool installation and model download.

Ollama was chosen as the local inference backend because it is the most mature open-source local LLM server with broad model support, a simple REST API, and built-in GBNF grammar enforcement. Other options (llama.cpp directly, LocalAI, GPT4All) were considered but Ollama offers the best developer experience and API surface.

## Alternatives Considered

| Alternative | Reason for Rejection |
|-------------|---------------------|
| GitHub Models API (Spec Kit approach) | Requires cloud API calls; violates local-first principle |
| OpenAI API directly | Cloud dependency; data leakage; per-token cost |
| Self-hosted vLLM on dedicated hardware | Much higher operational complexity; overkill for single-user workflow |
| Hybrid (local for generation, cloud for analysis) | Introduces cloud dependency for critical path; inconsistent availability |
| Anthropic API | Cloud dependency; data leakage concerns |

## Consequences

### Positive

- Complete operational independence from cloud LLM providers
- Zero ongoing API costs
- No data ever leaves the user's machine
- Works in air-gapped environments
- Predictable latency (no network variability)
- Single `pip install` / `uv tool install` for full system

### Negative

- Requires users to install and configure Ollama (additional setup step)
- Local models have lower quality than the largest cloud models for equivalent parameter counts
- Requires sufficient local hardware (8GB+ RAM minimum, 32GB+ recommended)
- Model download requires initial internet connectivity (5-40GB per model)
