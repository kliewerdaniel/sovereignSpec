# SovereignSpec Installation Guide

**Version 1.0.0**

---

## Prerequisites

### Required Software

| Component | Minimum Version | Installation Command |
|-----------|----------------|---------------------|
| Python | 3.11+ | `python3 --version` (bundled with macOS/Linux) |
| uv | 0.4+ | `curl -LsSf https://astral.sh/uv/install.sh \| sh` |
| Node.js | 18+ | `brew install node` (macOS) or `nvm install 18` |
| pnpm | 9+ | `npm install -g pnpm` |
| Git | 2.30+ | `brew install git` (macOS) or `apt install git` (Linux) |
| Ollama | 0.3+ | See below |

### Install Ollama

Ollama is the local inference backend. All SovereignSpec LLM calls go through Ollama's REST API at `localhost:11434`.

**macOS:**
```bash
curl -fsSL https://ollama.com/install.sh | sh
```

**Linux:**
```bash
curl -fsSL https://ollama.com/install.sh | sh
```

**Windows:**
Download the installer from [https://ollama.com/download](https://ollama.com/download) and run it.

**Verify installation:**
```bash
ollama --version
ollama serve  # Start the server (auto-starts as service on macOS/Linux)
```

### Recommended Ollama Models

Pull at least one model. The larger the model, the better the code generation quality.

| Model | Pull Command | RAM Required | Quality | Use Case |
|-------|-------------|-------------|---------|----------|
| Llama 3.1 70B | `ollama pull llama3.1:70b` | 48GB+ | Best overall | Complex reasoning, analysis, planning |
| Qwen 2.5 Coder 32B | `ollama pull qwen2.5-coder:32b` | 24GB+ | Best for code | Code generation, implementation |
| DeepSeek Coder V2 16B | `ollama pull deepseek-coder-v2:16b` | 16GB+ | Balanced | General SDD pipeline |
| Mistral 7B | `ollama pull mistral:7b` | 8GB+ | Minimum | Minimal hardware, basic specs |
| Nomic Embed Text | `ollama pull nomic-embed-text` | 2GB+ | N/A | Text embeddings (required for RAG) |

**Recommended minimum setup:**
```bash
ollama pull qwen2.5-coder:32b
ollama pull nomic-embed-text
```

### Verify Ollama Connectivity

```bash
curl http://localhost:11434/api/tags
```

Should return a JSON list of installed models.

## Installation Methods

### Method A: Via uv (Recommended)

```bash
uv tool install sovereignspec --from git+https://github.com/kliewerdaniel/sovereignSpec.git
```

This installs `sovereignspec` as a globally available command. Verify:

```bash
sovereignspec --version
# Output: sovereignspec 1.0.0
```

### Method B: Development Install

For contributors or users who want the latest unreleased changes:

```bash
git clone https://github.com/kliewerdaniel/sovereignSpec.git
cd sovereignSpec
uv sync
uv run sovereignspec --help
```

To make `sovereignspec` available globally from the dev install:

```bash
uv tool install --editable .
```

### Method C: Docker (Experimental)

```bash
docker pull sovereignspec/sovereignspec:latest
docker run --rm -v $(pwd):/workspace sovereignspec/sovereignspec init /workspace/my-project
```

Note: Docker requires Ollama to be accessible (mount host Ollama socket or use host networking).

## Verify Installation

Run the health check:

```bash
sovereignspec doctor
```

This checks:
1. **Python version**: Requires 3.11+
2. **Ollama connectivity**: Pings `localhost:11434/api/tags`
3. **Ollama models**: Verifies required models are pulled
4. **ChromaDB availability**: Confirms chromadb package is importable
5. **SQLite availability**: Confirms sqlite3 module is available
6. **File system permissions**: Checks write access to current directory

**Expected output (healthy system):**
```
✓ Python 3.11.7
✓ Ollama running (localhost:11434)
✓ Models: qwen2.5-coder:32b, nomic-embed-text
✓ ChromaDB available
✓ SQLite available
✓ File system writable
```

## Project Initialization

### Basic Initialization

```bash
# Create a new project directory with SovereignSpec structure
sovereignspec init my-project

# Initialize in the current directory
sovereignspec init .

# Force initialization in a non-empty directory
sovereignspec init . --force

# Specify Ollama model during init
sovereignspec init . --model qwen2.5-coder:32b

# Pre-configure an agent adapter
sovereignspec init . --integration claude-code

# Combine options
sovereignspec init my-api-project --model llama3.1:70b --integration opencode
```

### What Init Creates

```
my-project/
├── .sovereignspec/
│   ├── config.json              (Project configuration)
│   ├── bootstrap.md             (Universal agent bootstrap contract)
│   ├── constitution.md          (Project governing principles — initially empty)
│   ├── specs/                   (Your .sspec files go here)
│   ├── adr/                     (ADR-NNN.md files go here)
│   ├── tasks/                   (Task lists generated by compiler)
│   ├── patterns/
│   │   ├── pattern_library.json (Coding pattern library — auto-populated)
│   │   └── repository_map.json  (Repository intelligence map)
│   ├── memory/                  (Persistent storage)
│   │   ├── chromadb/            (ChromaDB vector store)
│   │   └── sovereignspec.db     (SQLite metadata database)
│   ├── graph/
│   │   └── graph.json           (Knowledge graph — adjacency list)
│   ├── agents/                  (Agent session records)
│   ├── grammar/                 (GBNF grammar files)
│   │   ├── spec_validation_result.gbnf
│   │   ├── implementation_plan.gbnf

│   │   ├── adr.gbnf
│   │   ├── test_case.gbnf

│   │   ├── adr-template.md
│   │   └── tasks-template.md
│   └── docs/                    (Auto-generated documentation)
└── (your project files)
```

## Integrating with Coding Agents

### Claude Code

```bash
sovereignspec integrate --agent claude-code
```

This writes:
- `CLAUDE.md` — Project-level instructions for Claude Code
- `.claude/commands/sovereign-constitution.md` — Command template
- `.claude/commands/sovereign-specify.md` — Command template
- `.claude/commands/sovereign-clarify.md` — Command template
- `.claude/commands/sovereign-plan.md` — Command template
- `.claude/commands/sovereign-tasks.md` — Command template
- `.claude/commands/sovereign-analyze.md` — Command template
- `.claude/commands/sovereign-implement.md` — Command template
- `.claude/commands/sovereign-checklist.md` — Command template

Claude Code automatically discovers commands in `.claude/commands/`. Run `/sovereign.*` commands in Claude Code's chat.

### OpenCode

```bash
sovereignspec integrate --agent opencode
```

This writes `AGENTS.md` with the full project context, agent contract, and `/sovereign.*` command definitions. OpenCode reads `AGENTS.md` on startup.

### Cursor

```bash
sovereignspec integrate --agent cursor
```

This writes `.cursor/rules/sovereignspec.mdc` with sovereignspec rules and command templates. Cursor applies rules automatically.

### Cline

```bash
sovereignspec integrate --agent cline
```

This writes `.clinerules` with the full SovereignSpec contract and command definitions. Cline reads this file on startup.

### RooCode

```bash
sovereignspec integrate --agent roocode
```

This writes `.roo/rules.md` with SovereignSpec mode configuration and command templates.

### Codex CLI

```bash
sovereignspec integrate --agent codex
```

This writes `AGENTS.md` (for Codex's file-awareness) and `skills/` directory with sovereignspec skill files.

### Windsurf

```bash
sovereignspec integrate --agent windsurf
```

This writes `.windsurfrules` with the full SovereignSpec contract.

### Continue

```bash
sovereignspec integrate --agent continue
```

This writes `.continue/config.json` with slash command configurations and `.continue/commands/` with command templates.

### Gemini CLI

```bash
sovereignspec integrate --agent gemini-cli
```

This writes `GEMINI.md` with the full SovereignSpec contract.

### Aider

```bash
sovereignspec integrate --agent aider
```

This writes `.aider.conf.yml` with SovereignSpec configuration and injects system prompt for SDD awareness.

### Generic (Universal Fallback)

```bash
sovereignspec integrate --agent generic
```

This writes only `.sovereignspec/bootstrap.md` — the universal contract. Any file-aware coding agent that reads the workspace can discover SovereignSpec through this file.

## Configuration Reference

### config.json Schema

```json
{
  "project": {
    "name": "my-project",
    "version": "1.0.0",
    "slug": "my-project"
  },
  "models": {
    "generation": "qwen2.5-coder:32b",
    "embeddings": "nomic-embed-text",
    "analysis": "llama3.1:70b"
  },
  "ollama": {
    "host": "http://localhost:11434",
    "timeout": 120,
    "stream": false
  },
  "adapter": {
    "type": "generic",
    "auto_detect": true
  },
  "graph": {
    "store": "adjacency-json",
    "neo4j": null
  },
  "chroma": {
    "path": ".sovereignspec/memory/chromadb",
    "distance": "cosine"
  },
  "sqlite": {
    "path": ".sovereignspec/memory/sovereignspec.db"
  },
  "watcher": {
    "enabled": false,
    "debounce_ms": 500,
    "watch_dirs": ["specs", "adr", "constitution.md"]
  },
  "templates": {
    "spec": ".sovereignspec/templates/spec-template.sspec",
    "adr": ".sovereignspec/templates/adr-template.md",
    "tasks": ".sovereignspec/templates/tasks-template.md"
  },
  "grammar_dir": ".sovereignspec/grammar",
  "specs_dir": ".sovereignspec/specs",
  "adr_dir": ".sovereignspec/adr",
  "tasks_dir": ".sovereignspec/tasks",
  "docs_dir": ".sovereignspec/docs",
  "patterns_dir": ".sovereignspec/patterns",
  "graph_path": ".sovereignspec/graph/graph.json",
  "bootstrap_path": ".sovereignspec/bootstrap.md",
  "constitution_path": ".sovereignspec/constitution.md"
}
```

### Field Descriptions

| Field | Type | Default | Description |
|-------|------|---------|-------------|
| `project.name` | string | (directory name) | Human-readable project name |
| `project.version` | string | "1.0.0" | Project version |
| `project.slug` | string | (name → slug) | URL-safe identifier |
| `models.generation` | string | "qwen2.5-coder:32b" | Model for code generation tasks |
| `models.embeddings` | string | "nomic-embed-text" | Model for text embeddings |
| `models.analysis` | string | (same as generation) | Model for analysis tasks |
| `ollama.host` | string | "http://localhost:11434" | Ollama API endpoint |
| `ollama.timeout` | integer | 120 | Request timeout in seconds |
| `ollama.stream` | boolean | false | Enable streaming responses |
| `adapter.type` | string | "generic" | Agent adapter type |
| `adapter.auto_detect` | boolean | true | Auto-detect agent from files |
| `graph.store` | string | "adjacency-json" | Graph storage backend |
| `graph.neo4j` | object/null | null | Neo4j connection config |
| `chroma.path` | string | "..." | ChromaDB persistence path |
| `chroma.distance` | string | "cosine" | Distance metric |
| `sqlite.path` | string | "..." | SQLite database path |
| `watcher.enabled` | boolean | false | Enable file watcher |
| `watcher.debounce_ms` | integer | 500 | Debounce interval for file events |
| `grammar_dir` | string | "..." | GBNF grammar files directory |
| `specs_dir` | string | "..." | .sspec files directory |
| `adr_dir` | string | "..." | ADR files directory |
| `tasks_dir` | string | "..." | Task files directory |
| `graph_path` | string | "..." | Knowledge graph file path |

## Troubleshooting

### Issue 1: "Ollama is not running"

**Symptoms:** `sovereignspec doctor` reports `✗ Ollama not reachable`

**Resolution:**
```bash
# Start Ollama
ollama serve

# Or restart the service
# macOS:
brew services restart ollama
# Linux:
sudo systemctl restart ollama
```

### Issue 2: "Model not found"

**Symptoms:** CLI reports `Model 'qwen2.5-coder:32b' not found`

**Resolution:**
```bash
# Pull the required model
ollama pull qwen2.5-coder:32b

# List installed models
ollama list
```

### Issue 3: "Insufficient memory for model"

**Symptoms:** Ollama crashes on model load, or CLI times out waiting for response

**Resolution:**
- Use a smaller model: `ollama pull mistral:7b`
- Set `"models": {"generation": "mistral:7b"}` in config.json
- Or increase system RAM (32GB minimum for 7B, 64GB+ for 32B)

### Issue 4: "Permission denied" when writing to .sovereignspec/

**Symptoms:** `sovereignspec init` fails with permission error

**Resolution:**
```bash
# Check directory ownership
ls -la .sovereignspec/
# Fix permissions
chown -R $(whoami) .sovereignspec/
```

### Issue 5: "ChromaDB import error"

**Symptoms:** `sovereignspec doctor` reports `✗ ChromaDB not available`

**Resolution:**
```bash
# Install chromadb
uv pip install chromadb

# Or install all dependencies
cd sovereignSpec && uv sync
```

### Issue 6: "SQLite database is locked"

**Symptoms:** Commands fail with "database is locked" error

**Resolution:**
- Ensure no other SovereignSpec process is running
- Delete the lock file: `rm .sovereignspec/memory/sovereignspec.db-journal`
- Or reset the database: `sovereignspec doctor --repair`

### Issue 7: "Spec validation hangs"

**Symptoms:** `sovereignspec spec validate --all` never completes

**Resolution:**
- Check Ollama is responding: `curl http://localhost:11434/api/tags`
- Reduce temperature in config: `"temperature": 0.1`
- Check for circular dependencies in specs

### Issue 8: "Graph file is corrupted"

**Symptoms:** CLI reports `✗ graph.json parse error`

**Resolution:**
```bash
# Backup corrupted file
cp .sovereignspec/graph/graph.json .sovereignspec/graph/graph.json.bak
# Regenerate from database
sovereignspec memory sync --rebuild-graph
```

### Issue 9: "Agent integration files not found"

**Symptoms:** Agent doesn't discover `/sovereign.*` commands

**Resolution:**
```bash
# Re-run integration
sovereignspec integrate --agent <agent-name> --force

# Verify files exist
ls -la CLAUDE.md .claude/commands/  # for Claude Code
ls -la AGENTS.md                    # for OpenCode/Codex
```

### Issue 10: "uv tool install fails"

**Symptoms:** `uv tool install sovereignspec` returns errors

**Resolution:**
```bash
# Ensure uv is up to date
uv self update
# Install from source
git clone https://github.com/kliewerdaniel/sovereignSpec.git
cd sovereignSpec
uv sync
uv tool install --editable .
```
